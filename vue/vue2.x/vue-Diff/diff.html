只要用到 虚拟 DOM 就一定会用到 Diff 算法 在比较的过程中，如果真的用 树形结构去比较，那么时间就是 O(n)的三次方，为了降低复杂度，需要引入 Diff 算法 js 执行 数据改变 -> 虚拟DOM(计算变更) -> 操作真实DOM -> 视图更新 那么什么是 虚拟DOM ---> 用 js 模拟DOM 结构 虚拟DOM 就是用老的虚拟DOM和新的去比较变化，计算出最小的需要更新的视图，然后再去更新视图 怎么找 就用 Diff 找 Diff 算法 1：遍历老的虚拟 DOM 2: 遍历新的虚拟 DOM 3: 更新排序 diff算法规则： 深度优先，同层比较 1：只比较同一层级，不跨级比较 2：比较标签名，标签名不同直接删除，不继续深度比较 3：key 相同，标签名相同，就认为是相同节点， diff中的 path 函数 打补丁 就是新旧比较 使用场景 1：首次渲染的第一个参数是不是真实的 DOM 元素 如果 是 就创建空的 Vnode 并且关联一个 DOM 元素，然后去 比较在 path 函数里传的第一个参数和第二个参数 是否是 同一个 Vnode 如果是用一个 就直接直接 path 函数进行更新 如果不是 就创建新的 DOM 元素 并且把老的 删除掉 当页面中多个 key 发生变化的时候，执行 diff 算法 进行新旧两次值的比较
